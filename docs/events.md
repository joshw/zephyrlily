# Lily Event Types

This document describes every event type a client will receive in the `"event"` WebSocket messages (and in the `/events` history replay). For each type it covers:

- Which `EventData` fields carry meaningful data
- The pre-formatted `text` field the proxy provides
- How the TUI overrides the display for message events
- What local state the client should update

See [client-api.md](client-api.md) for the overall API and the `EventData` structure.

---

## Pre-formatted text

Every event includes a `"text"` field containing a ready-to-display plain-text string generated by the proxy.  Clients can use this directly and apply their own styling without parsing the raw event fields.

```json
{ "id": 42, "type": "event", "data": {
    "event":  "rename",
    "source": "#23",
    "value":  "Alicia",
    "notify": true,
    "entities": { "#23": { "name": "Alice", ... } },
    "text":   "*** Alice is now named Alicia ***"
}}
```

**For simple events** (all events except `public`, `private`, `emote`, and `pa`): the `text` field is a complete, ready-to-display string.  No further parsing is needed ‚Äî apply your colour scheme and render it.

**For message events** (`public`, `private`, `emote`, `pa`): the `text` field contains a compact single-line summary.  Simple clients can display it as-is; rich clients (like the TUI) should use the raw fields to produce a more structured layout.

---

## Display conventions

- **`*** message ***`** ‚Äî third-party event; someone else did something.
- **`(message)`** ‚Äî self-confirmation; you performed the action.
- The proxy generates both forms, choosing based on whether `source == whoami`.
- Only display an event when `notify: true`.  The proxy forces `notify: true` for `public`, `private`, and `emote`.
- Only show a timestamp when `stamp: true`. Format as `HH:MM`.
- `recips` and `targets` entries are handles; look them up in the embedded `entities` map for names.
- Entity names are shown with blurb as `Name [blurb]` where the reference uses `%U` (presence/message events).

## Message events

These four events carry message content.  The proxy provides a compact summary in `text`; the TUI overrides it with a structured multi-line layout.

---

### `"public"` ‚Äî Public send

| Field | Content |
|-------|---------|
| `source` | Sender's handle |
| `recips` | Destination handles (discussions and/or users) |
| `value` | Message body |
| `time` | Send time (show when `stamp: true`) |

**`text` field (proxy plain-text summary)**
```
From Alice [busy] to emacs: hello everyone
From Alice [busy]: hello everyone           (no explicit recips)
```

**TUI display (rich override)**
```
 -> (14:23) From Alice [busy], to emacs:
 - hello everyone, wrapped at terminal width
```
The header uses distinct colours for ` -> `, the sender, and the ` - ` body prefix.  The body is word-wrapped; hard-broken tokens (e.g. URLs) carry no ` - ` prefix on their continuation lines.

**State updates** ‚Äî none.

---

### `"private"` ‚Äî Private send

| Field | Content |
|-------|---------|
| `source` | Sender's handle |
| `recips` | Recipient handles |
| `value` | Message body |

**`text` field**
```
Private from Alice [busy]: hey, got a minute?
```

**TUI display (rich override)**
```
 >> (14:24) Private message from Alice [busy]:
 - hey, got a minute?
```
Uses a different accent colour from public sends to make private messages immediately visually distinct.

**State updates** ‚Äî Track `source` as last private sender (`:` recall in intelligent-expand).  If `recips` has more than one entry, track sender + recips (excluding self) as last send-group (`=` recall).

---

### `"emote"` ‚Äî Emote action

| Field | Content |
|-------|---------|
| `source` | Sender's handle |
| `recips` | Destination handles |
| `value` | Action text |

**`text` field**
```
Alice waves hello
```

**TUI display (rich override)**
```
> (14:25, to emacs) Alice waves hello and
> wraps here if the line is long
```
The `> (timestamp, to dest) Name` header precedes the action text on the first line.  Continuation lines keep `> ` on word-boundary wraps; hard-break continuations (e.g. URLs) carry no prefix.

**State updates** ‚Äî Same tracking as `private`.

---

### `"pa"` ‚Äî Public address

| Field | Content |
|-------|---------|
| `source` | Sender's handle |
| `value` | Message text |

**`text` field**
```
Public address from Alice [admin]: server going down in 10 minutes
```

**TUI display (rich override)**
```
** Public address message from Alice [admin]: server going down in 10 minutes **
```

**State updates** ‚Äî none.

---

## Presence events

All presence events use `text` directly.

---

### `"connect"` ‚Äî User logged in

**`text`** (third-party / self handled the same ‚Äî connect is always third-party)
```
*** Alice [busy] has entered lily ***
```

**State updates** ‚Äî Set entity `state = "here"`.

---

### `"disconnect"` ‚Äî User logged out

| Field | Content |
|-------|---------|
| `value` | Reason string (may be empty) |

**`text`**
```
*** Alice [busy] has left lily (timeout) ***
*** Alice has left lily ***
```

**State updates** ‚Äî Set entity `state = "away"`.

---

### `"attach"` ‚Äî Session reattached

**`text`** ‚Äî `*** Alice [busy] has reattached ***`

**State updates** ‚Äî Set `state = "here"`.

---

### `"detach"` ‚Äî Session detached

| Field | Content |
|-------|---------|
| `value` | Reason (may be empty) |

**`text`** ‚Äî `*** Alice [busy] has been detached (reason) ***` / `*** Alice has detached ***`

**State updates** ‚Äî Set `state = "away"`.

---

### `"here"` ‚Äî Marked as present

**`text`** (third-party) ‚Äî `*** Alice is now "here" ***`

**`text`** (self) ‚Äî `(you are now here with the blurb [working on things])` / `(you are now here)`

**State updates** ‚Äî Set `state = "here"`.

---

### `"away"` ‚Äî Marked as away

| Field | Content |
|-------|---------|
| `value` | Non-empty when the server set away automatically (idled) |

**`text`** (explicit, third-party) ‚Äî `*** Alice is now "away" ***`

**`text`** (idled, third-party) ‚Äî `*** Alice has idled "away" ***`

**`text`** (self) ‚Äî `(you are now away with the blurb [working on things])`

**State updates** ‚Äî Set `state = "away"`.

---

### `"unidle"` ‚Äî User became active

**`text`** ‚Äî `*** Alice is now unidle ***`

Note: `notify` is typically `false` for self-unidle events.

**State updates** ‚Äî none.

---

## Identity events

### `"rename"` ‚Äî User renamed

| Field | Content |
|-------|---------|
| `value` | New name |

**`text`** (third-party) ‚Äî `*** Alice is now named Alicia ***`

**`text`** (self) ‚Äî `(you are now named Alicia)`

**State updates** ‚Äî Update entity `name`; update name-keyed index (remove old, add new).

---

### `"blurb"` ‚Äî Blurb changed

| Field | Content |
|-------|---------|
| `value` | New blurb (empty = cleared) |

**`text`** (third-party, set) ‚Äî `*** Alice has changed their blurb to [working on things] ***`

**`text`** (third-party, cleared) ‚Äî `*** Alice has turned their blurb off ***`

**`text`** (self, set) ‚Äî `(your blurb has been set to [working on things])`

**`text`** (self, cleared) ‚Äî `(your blurb has been turned off)`

**State updates** ‚Äî Update entity `blurb`.

---

### `"info"` ‚Äî Info file changed or cleared

| Field | Content |
|-------|---------|
| `recips` | Discussion handle when disc info changed; empty for personal info |
| `value` | Empty when the info was cleared |

**`text`** examples:
```
(you have changed the info for emacs)
(you have cleared the info for emacs)
(your info has been changed)
(your info has been cleared)
*** Alice has changed the info for discussion emacs ***
*** Alice has cleared the info for discussion emacs ***
*** Alice has changed their info ***
*** Alice has cleared their info ***
```

**State updates** ‚Äî none.

---

## Discussion events

### `"create"` ‚Äî Discussion created

| Field | Content |
|-------|---------|
| `recips` | Handle of the new discussion |

**`text`** (third-party) ‚Äî `*** Alice has created discussion emacs "The Emacs Discussion" ***`

**`text`** (self) ‚Äî `(you have created discussion emacs "The Emacs Discussion")`

**State updates** ‚Äî Add the new disc entity from `entities`.

---

### `"destroy"` ‚Äî Discussion destroyed

**`text`** (third-party) ‚Äî `*** Alice has destroyed discussion emacs ***`

**`text`** (self) ‚Äî `(you have destroyed discussion emacs)`

**State updates** ‚Äî Remove disc entity; remove from disc-membership set.

---

### `"join"` ‚Äî User joined a discussion

**`text`** (third-party) ‚Äî `*** Alice is now a member of emacs ***`

**`text`** (self) ‚Äî `(you have joined emacs)`

**State updates** ‚Äî If `source == whoami`: add each `recips` handle to the disc-membership set.

---

### `"quit"` ‚Äî User left a discussion

**`text`** (third-party) ‚Äî `*** Alice is no longer a member of emacs ***`

**`text`** (self) ‚Äî `(you have quit emacs)`

**State updates** ‚Äî If `source == whoami`: remove each `recips` handle from the disc-membership set.

---

### `"retitle"` ‚Äî Discussion title changed

**`text`** (third-party) ‚Äî `*** Alice has changed the title of emacs to "The Emacs Discussion" ***`

**`text`** (self) ‚Äî `(you have changed the title of emacs to "The Emacs Discussion")`

**State updates** ‚Äî Update disc entity `title`.

---

### `"drename"` ‚Äî Discussion renamed

Note: `source` is not meaningful. Disc names are prefixed with `-` per Lily convention.

**`text`** ‚Äî `*** Discussion -emacs is now named -gnu-emacs ***`

**State updates** ‚Äî Update disc entity `name`; update name-keyed index.

---

## Permission events

### `"permit"` ‚Äî Permission granted

| Field | Content |
|-------|---------|
| `recips` | Discussion handle |
| `targets` | Handle(s) receiving the permission |
| `sub_evt` | Privilege type (`"owner"`, role name, or empty for basic access) |

**`text`** ‚Äî selected examples:
```
(you have accepted ownership of discussion emacs)
(you have offered Alice ownership of discussion emacs)
(you have given Alice manager privileges to discussion emacs)
(emacs is no longer moderated)
*** Alice has offered you ownership of discussion emacs ***
*** Alice has given you manager privileges to discussion emacs ***
*** Alice has taken ownership of discussion emacs ***
*** Alice has given Bob manager privileges to discussion emacs ***
*** Alice has permitted you to discussion emacs ***
*** Alice has unmoderated discussion emacs ***
```

**State updates** ‚Äî none.

---

### `"depermit"` ‚Äî Permission revoked

Mirror of `permit`.

**`text`** ‚Äî selected examples:
```
(you have rescinded your offer to Bob for ownership of discussion emacs)
(you have removed Bob's manager privileges on discussion emacs)
(emacs is now moderated)
*** Alice has rescinded their ownership offer of discussion emacs ***
*** Alice has removed your manager privileges on discussion emacs ***
*** Alice has depermitted you from discussion emacs ***
*** Alice has moderated discussion emacs ***
```

**State updates** ‚Äî none.

---

## Appointment events

### `"appoint"` ‚Äî Role assigned

| Field | Content |
|-------|---------|
| `recips` | Discussion handle |
| `targets` | Handle(s) appointed |
| `sub_evt` | Role: `"owner"`, `"speaker"`, `"author"`, or generic |

**`text`** ‚Äî selected examples:
```
(you have accepted ownership of discussion emacs)
*** Alice is now the owner of discussion emacs ***
*** discussion emacs is now moderated ***
*** you have been made a speaker for discussion emacs ***
*** Bob is now a speaker for discussion emacs ***
*** you are now a moderator for emacs ***
```

**State updates** ‚Äî none.

---

### `"unappoint"` ‚Äî Role removed

**`text`** ‚Äî selected examples:
```
(you have rescinded your ownership offer to Bob of discussion emacs)
*** discussion emacs is no longer moderated ***
*** you are no longer a speaker for discussion emacs ***
*** Bob is no longer a speaker for discussion emacs ***
```

**State updates** ‚Äî none.

---

## Ignore events

### `"ignore"` ‚Äî Someone is ignoring you

**`text`** ‚Äî `*** Alice is now ignoring you ***` / `*** Alice is no longer ignoring you ***`

---

### `"unignore"` ‚Äî Someone stopped ignoring you

**`text`** ‚Äî `*** Alice is no longer ignoring you ***`

---

## Review event

### `"review"` ‚Äî Review cleared

**`text`** ‚Äî `*** Alice has cleared the review for discussion emacs ***`

---

## System events

### `"sysmsg"` ‚Äî Server system message

**`text`** ‚Äî `*** Server going down for maintenance at midnight ***`

---

### `"pa"` ‚Äî Public address

See [message events](#pa--public-address) above.

---

## Unknown event types

The `text` field is set to a neutral fallback:

```
[eventtype] #handle value
```
